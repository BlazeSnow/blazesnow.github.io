# <TitleIcon src="/icon/internet.svg" /> 前言

## 什么是远程链路？

在日常开发和使用中，许多场景需要从公网访问内网服务——远程办公、自建服务暴露、临时演示等。然而由于IPv4地址枯竭和NAT的广泛部署，大部分设备隐藏在局域网内，无法直接从外部访问。

远程链路技术通过在公网服务器与内网之间建立加密隧道，将流量安全地转发到目标设备，从而解决内网服务暴露的问题。

## 本章说明

本章将围绕公网服务器，给出两种形态的网络链路解决方案。

1. 部署在私有服务器的服务 -> 公网服务器 -> 用户
2. 部署在公网服务器的服务 -> 公网服务器 -> 用户

其中，①方案主要由frp实现；而②方案则较为丰富，主要由caddy、certbot实现。
